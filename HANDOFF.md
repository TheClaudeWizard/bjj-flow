# BJJ Flow — Project Handoff

## What This App Is

A single-file BJJ (Brazilian Jiu-Jitsu) training app at `/home/user/bjj-flow/`. It has three sections:
- **Flowchart** — interactive canvas graph for mapping techniques and transitions
- **Journal** — training session log
- **Calendar** — session calendar view

Everything lives in `index.html` (HTML/CSS/JS all in one file, ~1000 lines).

## What Has Been Done

### Phase 1 — PWA / iPhone Installable App
Converted to a Progressive Web App so it can be installed on iPhone via Safari "Add to Home Screen":

- `manifest.json` — PWA manifest (name, icons, display:standalone, theme colors)
- `sw.js` — Network-first service worker (serves fresh code when online, falls back to cache offline; old caches are purged on activate)
- `apple-touch-icon.png`, `icon-192.png`, `icon-512.png` — App icons generated by `generate-icons.py` (gold flat-top hexagon ring on dark background, pure Python stdlib)
- `index.html` `<head>` additions:
  ```html
  <meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover">
  <link rel="manifest" href="manifest.json">
  <link rel="apple-touch-icon" href="apple-touch-icon.png">
  <meta name="apple-mobile-web-app-capable" content="yes">
  <meta name="apple-mobile-web-app-status-bar-style" content="black">
  <meta name="apple-mobile-web-app-title" content="BJJ Flow">
  <meta name="theme-color" content="#1a1d26">
  ```
- Service worker registration at bottom of `<body>`:
  ```html
  <script>
  if('serviceWorker' in navigator){
    window.addEventListener('load',()=>navigator.serviceWorker.register('sw.js'));
  }
  </script>
  ```

**Update delivery**: push to `claude/load-bjj-index-file-bxv93` → GitHub Pages deploys → next time user opens the app on iPhone it fetches fresh files automatically.

### Phase 2 — Mobile UI Redesign (iPhone 11)

Replaced the desktop tab bar + toolbar with a proper mobile layout:

- **Top bar**: app title on the left, nav dropdown button on the right
- **Nav dropdown**: slides down from top-right with a dark overlay, contains Flowchart / Journal / Calendar links
- **Chart page layout**: flex column — canvas fills remaining space, toolbar pinned at bottom
- **Bottom chart toolbar**: mode buttons (Select / Add / Link / Group) + action buttons (Reset / Save / Load / Clear) in a single bar above the home indicator
- **Panels/modals** converted to bottom sheets: slide up from bottom with `transform: translateY(100%)` → `translateY(0)` transition
- `switchTab()` updated to reflect active tab in nav label and nav-item classes
- `setMode()` updated to use `mb-{k}` button IDs matching the new toolbar
- `env(safe-area-inset-*)` used throughout for notch/home-indicator clearance

### Phase 3 — Four iPhone UX Refinements

1. **Gesture discrimination** — Full touch handler rewrite. Introduced `touchMoved` flag (set after 6px movement threshold) and `hadMultiTouch` flag. Tap actions (select, add node, connect) only fire at `touchend` when both flags are false. Pan and pinch-zoom never accidentally activate chart elements.
   - Pinch zoom uses midpoint focal point math: `pan.x = px-(px-pan.x)*(newZoom/zoom)`
   - Two-finger pan is handled simultaneously with pinch

2. **Color legend moved** — From `left:12px; bottom:12px` to `right:12px; top:12px` (top-right, under the nav dropdown button).

3. **Toolbar raised** — Chart toolbar now clears the iPhone home-swipe area:
   ```css
   padding-bottom: calc(env(safe-area-inset-bottom, 0px) + 14px);
   height: calc(62px + env(safe-area-inset-bottom, 0px) + 14px);
   ```

4. **Retina / DPR canvas** — Canvas buffer now renders at physical pixel resolution:
   - `resize()`: `canvas.width = Math.round(W * dpr)` with `canvas.style.width = W+'px'`
   - `draw()`: `ctx.setTransform(dpr,0,0,dpr,0,0)` before every `clearRect` — all coordinate logic stays in CSS pixels
   - `drawGraph()` (the calendar graph): same DPR treatment applied

## Key Architectural Notes

- **All app code is in `index.html`** — no build step, no npm, no framework.
- **Canvas coordinate system**: pan/zoom state is `{pan: {x,y}, zoom}`. Helper functions `wx(px)` and `wy(py)` convert screen coords to world coords. DPR scaling is applied only in the canvas transform, so all business logic uses CSS pixel coordinates.
- **Data persistence**: `localStorage` via `saveData()` / `loadData()`. The save/load buttons in the toolbar also support JSON file export/import.
- **Touch vs mouse**: touch events use `e.preventDefault()` (passive:false) to block browser scroll/zoom. Mouse events handle desktop use.
